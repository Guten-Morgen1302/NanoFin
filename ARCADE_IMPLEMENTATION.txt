ğŸš¨ IMPORTANT NOTES

1. PUBLIC/GAMES FOLDER
   The games must be in public/games/ for the dev server to serve them.
   Run setup-arcade.bat or npm run setup:arcade before npm run dev.

2. IFRAME SECURITY
   Games are served from the same domain, so CORS is not an issue.
   Each game runs in isolated iframe for security.

3. GAME LOGIC
   Games maintain their own state and logic. No backend sync by default.
   Optional: Implement REST API to save game progress to server.

4. OFFLINE CAPABILITY
   Games work completely offline once cached by browser.
   Training scores persist locally via localStorage.

5. MOBILE RESPONSIVE
   Tested for mobile (see CSS media queries).
   Games may need scroll/zoom on small screens.
   Consider adding landscape lock for mobile gaming.

âœ‰ï¸ SUPPORT & TROUBLESHOOTING

Common Issues:

Q: "Games not loading in modal"
A: 1) Check public/games/ exists with proper structure
   2) Run setup-arcade.bat
   3) Clear browser cache (Ctrl+Shift+R)
   4) Check browser console for errors

Q: "Behavior score not updating"
A: 1) Verify localStorage is enabled in browser
   2) Check DevTools > Application > Local Storage
   3) Games must run onGameComplete callback
   4) Try clearing app state and reloading

Q: "Modal won't close"
A: 1) Click the âœ• button in top-right
   2) Press ESC key
   3) Click outside (backdrop)
   4) Check console for JS errors

Q: "Game lag or slow performance"
A: 1) Check if running on older device
   2) Reduce other browser tabs
   3) Check browser CPU/memory usage
   4) Try different browser
